# GitHub Upload Guide

## Recommended repository name

```text
access-remedies-cross-border-mergers-network-industries
```

## Recommended visibility

- Use **Private** if you want to keep the working paper confidential while maintaining version control.
- Use **Public** if you want a public authorship timestamp and public working-paper visibility.

## Upload through GitHub website

1. Log in to GitHub.
2. Click **New repository**.
3. Use the repository name above.
4. Do not choose an open-source license from GitHub's default options.
5. Create the repository.
6. Upload the unzipped contents of this package.
7. Commit with the message:

```text
Initial working paper package
```

## Create a release

1. Open the repository.
2. Click **Releases**.
3. Click **Draft a new release**.
4. Use this tag:

```text
v0.1-working-paper
```

5. Use this release title:

```text
Initial Working Paper Draft
```

6. Upload the ZIP package as a release asset.
7. Publish the release.
